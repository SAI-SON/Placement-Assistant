8.2  UPLOAD RESUME  ( /upload-resume )
════════════════════════════════════════════════
FILE: src/app/upload-resume/page.tsx

  STATE VARIABLES:
    resumeFile       : File | null
    isPending        : boolean (useTransition)
    analysis         : ResumeScoringAndFeedbackOutput | null
    learningSuggestions : { videoSuggestions: string[] } | null

  USER FLOW:
    STEP 1 — UPLOAD VIEW (shown when analysis === null):
      Left side — Drop zone card:
        - react-dropzone handles drag/drop
        - Accepted: application/pdf (.pdf), .docx (max 4MB)
        - Validation:
            file.type !== 'application/pdf' AND !== docx → toast error
            file.size > 4MB → toast error
        - When file selected: shows CheckCircle2 icon + filename
        - When dragging: bg-primary/20 border-primary, text "Drop it like it's hot!"
        - Analyze button: disabled if no file or pending
          bg-primary hover:bg-primary/90 hover:scale-105, rounded-lg text-lg py-6

      Right side — Features list:
        3 bullet points with icon+text:
          Search icon  → "Keyword Match"
          BarChart2    → "Impact & Metrics"
          FileCheck    → "Formatting & Readability"

    STEP 2 — ANALYSIS (server action called):
      1. File read as base64 DataURI (FileReader.readAsDataURL)
      2. analyzeResumeAction({ resumeDataUri }) called
      3. If score < 70 → also calls getLearningSuggestionsAction
      4. Success toast: "Analysis Complete!"

    STEP 3 — RESULTS VIEW (shown when analysis !== null):
      Section A — Score Card:
        Glassmorphism card with <ScoreChart score={analysis.score} />
        Plus summary text: analysis.summary

      Section B — Two-column grid:
        Left card  — "What We Loved" (ThumbsUp icon text-green-400)
          List items prefixed with ✓ in text-green-300
        Right card — "What's Holding You Back" (AlertTriangle text-yellow-400)
          List items prefixed with ! in text-yellow-300

      Section C — "Your Quick-Win Checklist":
        Each improvement item has:
          Badge (Critical/High Impact/Suggestion) based on title keywords
          Title (stripped of parenthetical)
          "What's Wrong:" label + text
          "How to Fix:" label + monospace code block (bg-muted/30 p-3 rounded-md)
          Card styling by severity:
            Critical    → bg-red-900/20    border-red-500/30
            High Impact → bg-yellow-900/20 border-yellow-500/30
            Suggestion  → bg-green-900/20  border-green-500/30

      Section D — "Personalized Learning" (only if score < 70):
        YouTube icon (text-primary)
        List of video title suggestions → links to YouTube search results
        Each link: Youtube icon + title + ExternalLink icon

      Footer:
        "Analyze Another Resume" button (variant=secondary) → resets all state

SECTION 9: API ROUTES
--------------------------------------------------------------------------------

  GET /api/news
  FILE: src/app/api/news/route.ts
    Query params: state (required), district (required)
    Reads: process.env.NEWS_API_KEY

    Returns: raw NewsAPI response (articles array)
    Errors: 400 if missing params, 500 if no API key, relay status if NewsAPI error

  GET /api/youtube-search
  FILE: src/app/api/youtube-search/route.ts
    Query params: topic (required)
    Reads: process.env.YOUTUBE_API_KEY
    Endpoint: https://www.googleapis.com/youtube/v3/search
    Params: part=snippet, type=video, maxResults=10, q="{topic} tutorial"
    Maps response items to: { id, title, channel, thumbnail, url }
    Returns: { videos: Video[] }
    Errors: 400 if missing topic, 500 if no API key, relay status on YouTube error

SECTION 14: KNOWN BUGS & THEIR EXACT FIXES
--------------------------------------------------------------------------------

  BUG 1 — Turbopack + Google Fonts
    ERROR : Module not found: @vercel/turbopack-next/internal/font/google/font
    ROOT  : --turbopack flag incompatible with next/font/google (next@15)
    FIX   : package.json: "dev": "next dev -p 9002" (remove --turbopack)

  BUG 2 — Deprecated Next.js Image API
    ERROR : layout="fill" and objectFit="cover" crash at runtime
    ROOT  : These props were removed in Next.js 13+
    FIX   : <Image fill unoptimized className="object-cover opacity-80"/>
    FILES : src/app/job-news/page.tsx (line 126)
            src/app/career-pathway/page.tsx (line 154) — same fix needed

  BUG 3 — @genkit-ai/firebase missing module
    ERROR : Module not found: Can't resolve '@genkit-ai/firebase'
    ROOT  : @genkit-ai/core/lib/tracing.js tries to load optional Firebase dep
    FIX   : next.config.ts serverExternalPackages + webpack alias = false

  BUG 4 — Wrong Gemini model (gemini-2.5-flash)
    ERROR : fetch failed / model not found
    ROOT  : gemini-2.5-flash not publicly released
    FIX   : Change to 'googleai/gemini-2.0-flash-lite' in genkit.ts

  BUG 5 — Wrong Gemini model (gemini-2.0-flash)
    ERROR : [429] Quota exceeded — limit: 0 for free_tier_requests
    ROOT  : gemini-2.0-flash requires billing; free tier quota = 0
    FIX   : Change to 'googleai/gemini-2.0-flash-lite' (has free tier)

  BUG 6 — Corrupted .env.local
    ERROR : ConnectTimeoutError — API key not being sent
    ROOT  : GOOGLE_GENAI_API_KEY had literal text "npm install @google/generative-ai"
    FIX   : Rewrite entire .env.local with correct keys

  BUG 7 — API key not explicitly passed
    ERROR : Silent failure — googleAI() can't find key in some environments
    ROOT  : googleAI() without explicit apiKey relies on env var lookup order
    FIX   : googleAI({ apiKey: process.env.GOOGLE_GENAI_API_KEY })
